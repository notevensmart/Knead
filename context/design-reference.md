# Design Reference

Primary Design Standard:
Use the installed Codex UI/UX skills as the main reference for all design work in this repository.

Scope:
The Codex UI/UX skills should guide:
- Visual hierarchy
- Layout and spacing
- Responsive behaviour
- Accessibility
- Navigation patterns
- Interaction states
- Typography choices
- Colour usage
- Asset treatment
- Overall prototype polish

Content Source Of Truth:
Use context/extracted-requirements.md for all client-specific facts, required pages, required sections, services, contact information, social links, brand personality, and constraints.

Design Direction For Knead:
- Calm, professional, peaceful, relaxed, and accessible.
- Minimal text with strong clarity and emotional softness.
- Artful enough to feel memorable, but not busy or intimidating.
- Use the logo palette as the base: warm gold, soft yellow, pale mint green, black, and white.
- Feature the real Knead logo and team portraits where useful.

Final Site Expectations:
- The client selected Prototype 1 as the final direction.
- The root site should include About, Services, and Contact.
- Contact can be part of the main page.
- Services should be listed without invented detailed descriptions.
- Include a clear connect call to action.
- Do not invent a phone number.
